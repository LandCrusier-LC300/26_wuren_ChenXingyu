4.在测试过程中，我发现测试的损失图像与视频教程中的图像很不一致，虽有一点下降的趋势，但在中间处总会有尖顶出现。针对于此，我使用豆包进行了研究，问题分析如下：
1. 「Batch 数据本身的差异」（最常见）
训练时每个 Batch 的数据分布不均匀，比如：

    部分 Batch 里的样本本身就很难（比如异常样本、边缘场景、噪声大的样本），模型在这些 Batch 上计算的 Loss 自然会突然升高。
    数据增强策略在部分 Batch 上产生了 “难样本”，导致 Loss 波动。
    数据集中存在标签错误、标注噪声，这些样本集中出现在某些 Batch 中，会直接造成 Loss 尖峰。

2. 「优化器 / 超参数的问题」

    学习率过高：如果学习率设置得太大，模型参数更新的步长就会很大，容易在局部最优附近来回震荡，导致 Batch Loss 剧烈波动。
    Batch Size 太小：小 Batch 对数据分布的估计本身就不稳定，每个 Batch 的梯度噪声很大，Loss 曲线自然会 “上下跳”。
    梯度爆炸 / 消失：如果模型结构设计不当（比如深层网络没有残差连接、激活函数选择不当），可能出现梯度异常，导致 Loss 突然飙升。

3. 「训练流程的问题」

    没有梯度裁剪：当某些 Batch 的梯度突然变大时，参数更新会直接 “冲过头”，造成 Loss 尖峰。
    学习率调度器失效：比如没有使用学习率衰减，或者衰减策略设置不合理，后期学习率仍然过高，导致 Loss 无法稳定。
    数据加载问题：比如数据打乱不充分，导致同分布的样本集中出现在某些 Batch 中，造成 Loss 波动。

回答中显示主要问题出现在样本上，对训练样本进行观察，我发现有部分图片存在标注错误的问题，可能是损失图像问题产生的原因。
二。基础作业二：
1.深入研究有关概念：
  卷积神经网络：在概率统计中学习到的卷积是一个积分，在这里就变成了一个卷积核在图像上滑移运算，通过AI得知，二者本质上是一致的；在代码撰写过程中，我发现卷积层将3个通道/特征（RGB）变为多个特征，本质上是通过不同卷积核提取多个特征。最后又变为3个特征，这三个特征属于模型训练过程中通过特征提取和学习所提炼出图片中所明显显示出的三个特征（三个颜色）。
  反向传播：该步骤是在一次前向传播后进行的。可以理解为中小学生的学习：白天在课堂上第一次学习知识，属于前向传播；晚上再进行复习和做作业，属于反向传播。我的理解就是让模型对第一次学习结果进行复盘，看那些没有学到位，那些没有真正学懂，从而进行针对性的参数修改，保证模型训练的精准度。
  激活函数：ppt里面写的是对卷积层的输出进行变换，增加网络的非线性表达能力。就像算术一样，没有激活函数，就像只能进行加减乘除运算，叠加在多层可能也不过是使用一些结合律，分配率或者巧妙算法，不能算积分，微分等高阶算法。给模型训练加上激活函数，我觉得用“如虎添翼”这个词形容比较好。
2。了解有关深度学习的知识：
yolo:我对yolo的理解是CNN的拓展，相当于CNN+决策，能够分辨图像中的东西是什么。从原理角度来讲，将图像按网格分为不同的部分，每个网格都对框住的内容进行识别，可以一次性判别出图像中有哪些东西。典型应用就是垃圾分类，直接判别垃圾种类，便于控制投放方向。
3.对深度学习的认知：之前觉得做深度学习是很复杂的事情，但在经历了一次代码撰写后，发现绝大多数地层的东西都被写入库中了，训练模型时只需直接调用。尽管如此，深度学习模型的训练依然复杂，需要进行神经网络的搭建，这就涉及到数据集的获取与处理，网络参数的设定等等。我觉得要想真正掌握，需要多写代码，多经历，多看，多理解。
